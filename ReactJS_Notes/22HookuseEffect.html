useEffect() | React

In React, useEffect is one of the most essential Hooks used to manage side effects in functional.

Side Effecs are operations that occur outside of the component's render cycle, such as data fetching, subscriptions, or manually interacting with the DOM.

The useEffect hook allow you to Perform these side effects after the component has rendered.

useEffect() | Effect Function:

The first argument of useEffect is the effect function. this function is called after the component has rendered, and it's where you place the code for your sode effect.

it can be an asynchronous function, and you can perform any asynchronous opesations like data fetching inside it.

useEffect() | Dependency Array:

useEffect(, )

The second argument of useEffect is an array of dependencies. This array tells React when ti re-run the effect. and its opational

You can include to control when the effect should be executed.

useEffect() |Based On Dependency

1. If the dependency array is empty([]), the effect will only run once, after the initail render

2. If the dependency array is not provided the effect will run after every render of the component.

3. If the dependency array contains variables, the effect will only be re-executed when any of those variables change.
