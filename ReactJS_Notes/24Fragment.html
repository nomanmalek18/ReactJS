React Fragments

React fragment: Group multiple elements without adding extra DOM wrappers. Useful to return multiple elements from a component's render method wothout extra containers

with React 16.2 and about Fragments you can use a fragment to avoid adding the extra wrappers, Fragments are denoted by the special syntax:

<React.Fragment> or
<> </> the shorthand

without Fragments

Prior ti React 16.2 you had to wrap them in a single pareny element.

For Example:

import React, from "react"; const MyComponent() => { return ( <div> // Wrapper <h1>Hello</h1> <p>Thid is a paragraph.</p> </div> ); }; export defaule MyComponent;

With Fragments

A fragment is simply an empty element that acts as a container for other elements without rendering any additional DOM nodes.

1 2 3 4 5 6
import React, from "react";

const MyComponent() => {
return (
<> // React Fragment Wrapper
<h1>Hello</h1>
<p>Thid is a paragraph.</p>
</>
);
};
export defaule MyComponent;

With Fragments

Fragments can also have a key prop, just like any other React element. the key prop os useful when you're dynamically rendering lists of fragment. with >React.Fragment>

1 2 3 4 5 6
import React, from "react";

const MyComponent({items}) => {
return (
<>
{items.map(()=>(
<React.Fragment key={item.id}>
<span>{item.name}</span>
<br />
</React.Fragment>
))}
</>
);
};
export defaule MyComponent;

If not using Fragments

Extra DOM nodes:

Using a regular container introduces additional DOM nodec in the rendered output, which can lead to increased memory usage and slightly slower rendering.

CSS Conflicts:

It Wrapper element has its own CSS styles or classes, they might unintentionally affect the layout or styles of the child elements inside the component.

Semantic impact:

sometimes, adding extra wrapper elements can change the Semantic structure of the document which might not be desirable from an accessibility standpoint.

Nesting depth:

Repeatedly nesting element within multiple container may make the component tree deeper potectially affecting performanse and readability.
